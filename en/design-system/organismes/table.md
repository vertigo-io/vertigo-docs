# Table

A table is an excellent way to present a lot of information in a limited space.
They are very useful for densifying the display, allowing users to visualize a lot of information quickly without "scrolling" or navigating.
Tables also make it easy to compare multiple rows and are therefore facilitators for your users' work.
They generally have pagination and sorting functions.
It is also possible to propose actions on each row in a way that is contextual to the element.

We chose to make tables look similar to blocks so they can be placed near (generally above or below) a block or another table.
They can therefore have an optional title as well as actions at the top right.

In some cases, it may be useful to be able to select certain rows; in this case, a selection checkbox is present in a dedicated first column.


# Best Practices

- Prefer left alignment for all textual data
- Prefer right alignment for all numeric data
- Limit the number of available actions at the top right (<3); for complex actions, prefer navigation to a dedicated screen
- Limit the number of available actions per row (<3); for complex actions, prefer navigation to a dedicated screen
- Prefer displaying row actions on hover to avoid overloading the display with redundant elements.
- Tables are better suited for applications used primarily on desktop computers
