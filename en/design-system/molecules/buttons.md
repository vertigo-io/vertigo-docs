# Buttons

Buttons allow a user to perform actions. Their distinctive shape draws the user's attention and guides them through their journey.

Their placement in a layout should maximize the fluidity of the user journey. Buttons should therefore be sorted by order of importance or relevance, and care should be taken to make them easily accessible.
Buttons can vary in size, color, and shape depending on the situation.

We have selected three types of buttons:

 - round buttons with icon
 - rectangular buttons with label
 - rectangular buttons with label and icon

When many actions are available, they should be placed in a group. This group, depending on available space, will display the main actions directly and place the others in an "Other actions" category.

# Best Practices

- Ensure that the button label or icon unambiguously indicates the corresponding action
- The label should be as short as possible and always on a single line
- Highlight the most important actions using the primary color
- Within the same area of the page, do not mix different types of buttons (round, rectangular) and always use the same size
- Default active buttons should always correspond to risk-free actions (e.g.: cancel, back, close...)
